Code Artifacts

The Hospital Management System evolved from a basic console application to a complete full-stack web application through systematic enhancements in software design, algorithms, and database integration. This page provides access to both the original code (pre-enhancement) and the enhanced code (final version) demonstrating the full scope of improvements implemented during the capstone project.

Project Evolution

The transformation of the Hospital Management System represents a comprehensive application of computer science principles across three major enhancement categories. Each phase built upon the previous work, creating a cohesive, professional-quality application suitable for production deployment.

Original Code

Console-Based Java Application

The original artifact from earlier coursework included:

  • Fixed-size arrays for data storage (limited to 50 items per entity)
  • Basic CRUD operations with linear search
  • No persistence (all data in-memory only)
  • Simple console menu interface
  • Monolithic architecture with tightly coupled components
  • Minimal validation and error handling

Limitations:

  • Data lost when program closes
  • Cannot scale beyond fixed capacity
  • O(n) search time for all operations
  • No appointment conflict detection
  • Unsuitable for production use

System Architecture

Full-Stack Integration

The complete system demonstrates end-to-end application development with three tiers:

Presentation Layer (React Frontend)

User interacts with dashboard at http://localhost:3000

Forms submit data, charts display visualizations

JWT tokens stored in browser for authentication

Application Layer (Spring Boot Backend)

REST API running at http://localhost:8080

Controllers handle HTTP requests

Services contain business logic and validation

Repositories abstract database operations

Data Layer (MongoDB Database)

MongoDB server running at mongodb://localhost:27017

Database: gms_db

Collections: doctors, patients, appointments, employees

Indexed queries for fast retrieval

Key Features Implemented

Security Features

  • JWT Authentication: Secure token-based authentication protecting all API endpoints
  • BCrypt Password Hashing: One-way encryption with automatic salt generation
  • CORS Configuration: Properly configured cross-origin resource sharing
  • Input Validation: Comprehensive validation at both frontend and backend
  • Protected Routes: Role-based access control foundation
  • Session Management: Stateless JWT tokens with expiration

Functional Features

  • Doctor Management: Add, update, delete, and search doctors by specialty
  • Patient Management: Maintain patient records with demographic information
  • Appointment Scheduling: Create appointments with conflict detection
  • Employee/Payroll: Track employees and calculate payroll based on appointments
  • Dashboard Analytics: Real-time statistics and data visualization
  • Search Functionality: Quick lookup by name, specialty, or ID

Technical Features

  • Scalable Architecture: Repository pattern allows easy database switching
  • Optimized Algorithms: O(n log n) performance for scheduling operations
  • Data Persistence: All data stored permanently in MongoDB
  • RESTful Design: Standard HTTP methods and status codes
  • Responsive UI: Works on desktop, tablet, and mobile devices
  • Error Handling: Graceful handling of exceptions with user-friendly messages

Running the Application

Prerequisites

  • Java JDK 17 or higher
  • Node.js 16+ and npm
  • MongoDB 8.0 (or MongoDB Atlas)
  • Maven 3.6+
  • Git

Setup Instructions

1. Start MongoDB

mongod --dbpath="C:\data\db"

2. Start Backend (Terminal 1)

cd hospital-backend
mvn spring-boot:run

Backend runs at: http://localhost:8080

3. Start Frontend (Terminal 2)

cd hospital-dashboard
npm install
npm start

Frontend runs at: http://localhost:3000

4. Login

Username: admin

Password: admin123
